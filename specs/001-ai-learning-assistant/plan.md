# Implementation Plan: AI Learning Assistant Platform

**Branch**: `001-ai-learning-assistant` | **Date**: 2026-06-02 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/001-ai-learning-assistant/spec.md`

## Summary

Build a production-ready multilingual AI Voice and Chat Assistant for educational
institutes. End-users (students, parents, visitors) interact via voice or text to
discover courses, receive personalised recommendations, and optionally submit
contact details. A non-technical admin panel allows institute staff to manage all
educational content (courses, batches, FAQs, announcements); the AI assistant
reflects all changes immediately without redeployment.

**Technical approach**: Next.js 14 (App Router) frontend on Vercel; OpenAI Agents
SDK for AI orchestration and tool-calling against Supabase; OpenAI Realtime API
for voice (ephemeral-token relay pattern); Supabase PostgreSQL + RLS for all
persistent data; Supabase Auth for admin authentication.

## Technical Context

**Language/Version**: TypeScript 5.x (frontend and Edge Functions); Deno 1.x (Supabase Edge Functions)
**Primary Dependencies**: Next.js 14 (App Router), Tailwind CSS v3, shadcn/ui, OpenAI Agents SDK (beta), OpenAI Realtime API (WebSocket), Supabase JS v2, Supabase Auth, Supabase Edge Functions
**Storage**: Supabase PostgreSQL (single source of truth); no secondary stores
**Testing**: Playwright (E2E вЂ” voice/text flows, admin panel); Vitest (unit вЂ” agent tools, validators)
**Target Platform**: Web вЂ” Vercel (frontend) + Supabase Cloud (database, auth, edge functions); modern browsers (Chrome 112+, Safari 16+, Firefox 113+)
**Project Type**: Web application (Next.js frontend + Supabase backend)
**Performance Goals**: LCP < 2.5 s on 4G; voice response latency < 500 ms end-to-end; admin page load < 1 s
**Constraints**: No client-side secrets; RLS on every table; mobile-first (375 px+); WCAG 2.1 AA; graceful degradation to text-only when microphone unavailable
**Scale/Scope**: 50 concurrent sessions (SC-008); 9 DB entities; 5 user stories; single-institute deployment for MVP

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-checked post-design.*

### Pre-Phase 0 (initial)

- [x] **Principle I вЂ” Dynamic Content First**: All course/FAQ/announcement data
  retrieved via AI agent tool calls to Supabase at runtime. Zero hardcoded
  educational content in prompts or components.
- [x] **Principle II вЂ” Multilingual By Default**: Language detection in agent
  system prompt + Realtime API voice session. Applies to text and voice equally.
- [x] **Principle III вЂ” Voice-First**: OpenAI Realtime API for voice; text chat
  always available; both channels expose identical AI capabilities.
- [x] **Principle VI вЂ” Zero Client Secrets**: OpenAI API key and Supabase service
  role key are server-only. Browser receives only ephemeral tokens with scoped
  permissions.
- [x] **Principle VII вЂ” Scalability Without Code Changes**: All educational
  entities are database rows. Adding a new course type requires only Supabase
  data operations.

### Post-Phase 1 (re-check after design)

- [x] Data model is fully dynamic вЂ” no entity type is hardcoded.
- [x] API contracts expose no server-side secrets to callers.
- [x] RLS policies drafted for every table (see data-model.md).
- [x] All user-facing strings are passed through the AI response layer (multilingual).

**Structure Decision**: Web application pattern. Next.js App Router handles both
the public chat UI and the admin panel in a single deployment. Server-side API
routes (`app/api/`) act as the secure proxy between the browser and
OpenAI/Supabase service-role credentials. No separate backend service is needed
for MVP вЂ” Supabase Edge Functions are reserved for compute-intensive offloads
that emerge post-launch.

## Phase 0 Research Summary

See [research.md](research.md) for full findings. Key decisions:

| Area | Decision | Rationale |
|---|---|---|
| Voice transport | Ephemeral token relay (not WebSocket proxy) | Lower latency; browser connects directly to OpenAI after receiving a scoped token from `/api/voice/session` |
| AI grounding | Agent tool calls at inference time | Satisfies Dynamic Content First; no RAG index required for MVP |
| Language detection | GPT-4o intrinsic detection via system prompt instruction | No separate NLP service needed; handles EN/UR/Roman UR natively |
| Admin auth | Supabase Auth middleware + Next.js `middleware.ts` | Centralised session management; no custom JWT logic |
| RLS strategy | Anonymous public read on courses/FAQs; admin-only write; leads insert-only for anon | Minimal privilege per role |
